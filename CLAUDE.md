## Что это за проект

Telegram Mini App — персональный AI-астролог. Пользователь вводит дату, время
и место рождения. Получает натальную карту, ежедневные прогнозы и живой чат
с астрологом который его "помнит". Фокус на отношениях — синастрия (совместимость пар).

**Слоган:** «Личный астролог. Всегда на связи.»

**Аудитория:** Женщины 22–42 лет, Россия и СНГ, интерес к саморазвитию и отношениям.

**Важно:** Сервис развлекательный. Прогнозы не являются научными предсказаниями.
Это обязательно указывается в условиях использования.

  synastry_json, created_at

## Логика проверки доступа (access.js)

```javascript
const LIMITS = {
  free:     { messages: 5,  partners: 1,  forecastDays: 1  },
  trial:    { messages: 30, partners: 5,  forecastDays: 7  },
  pro:      { messages: 30, partners: 5,  forecastDays: 7  },
  platinum: { messages: 80, partners: 10, forecastDays: 30 },
}

const FEATURES = {
  transits_basic:  ['pro', 'platinum'],
  transits_full:   ['platinum'],
  progressions:    ['platinum'],
  solar_included:  ['platinum'],
  notifications:   ['pro', 'platinum'],
  pdf_export:      ['platinum'],
  sonnet_priority: ['platinum'],
}

async function checkAccess(userId, feature) {
  const user = await getUser(userId)
  const plan = getActivePlan(user) // учитывает триал и срок подписки

  // Подписочные фичи
  if (FEATURES[feature]) {
    return FEATURES[feature].includes(plan)
  }

  // Лимит сообщений
  if (feature === 'chat_message') {
    return user.messages_used_this_month < LIMITS[plan].messages
  }

  // Разовые покупки
  if (feature.startsWith('purchase_')) {
    const productType = feature.replace('purchase_', '')
    return await hasPurchase(userId, productType)
  }

  return false
}
```

## Как работает AI-астролог

Claude API не хранит данные между сессиями.
При каждом сообщении пользователя:

```
1. Пользователь пишет вопрос
2. Backend достаёт из Supabase:
   - натальную карту пользователя
   - текущие астрологические транзиты
   - последние 20 сообщений чата
   - данные партнёра (если есть)
3. Всё упаковывается в один промпт
4. Claude генерирует персональный ответ
5. Ответ сохраняется в chat_history
```

### Модели Claude API

```
claude-haiku-4-5   → прогнозы дня, synastry_quick, простые вопросы
                     Дешевле в 5 раз. Быстро.

claude-sonnet-4-6  → сложный чат, PDF-отчёты, Платинум-пользователи
                     Качественнее. Всегда для платных разборов.
```

### Оптимизация расходов API

```
✅ Кэшировать прогнозы — раз в сутки, не при каждом запросе
✅ 12 базовых шаблонов по знаку Луны (экономия 60–70%)
✅ Haiku для простых запросов и synastry_quick
✅ Sonnet только для PDF и сложных вопросов
✅ Лимиты сообщений по тарифам
```
